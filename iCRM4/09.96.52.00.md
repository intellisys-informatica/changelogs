:warning: **Mensagem de erro de login**

Não existia tratamento para quando o usuário se encontrar em determinadas situações exibir as devidas mensagens de erro. Agora o sistema exibe as mensagens corretamente para os seguintes casos:

- **Usuário inativo:** Informa ao usuário que sua conta está inativa.
- **Dados não conferem:** Informa que o login ou senha estão incorretos.
- **Usuário não cadastrado:** Informa que o usuário não existe no sistema.

---

:warning: **Salvando dados da proposta ao aprovar proposta**

Os dados da proposta só estavam sendo salvos quando o operador clicava em **"Confirmar"**. Agora o botão **"Aprovar"** também salva as informações no banco de dados caso o operador altere alguma informação na tela e clique diretamente nele, sem a necessidade de passar pelo passo de confirmação.

---

:star: **Integração com o iBank**

Agora os boletos do sistema podem ser gerados e enviados ao banco diretamente via API, sem necessidade de arquivo de remessa manual.

**Bancos homologados:**
- Itaú
- Banco do Brasil

A integração abrange os módulos de fatura (Private e Convênio) e cobrança de parcelas. O controle de envio é realizado pela nova tabela `TB_INTEGRACAO_BOLETO`, que registra o status de cada boleto integrado. Além disso, suporte a **PIX** foi adicionado nas tabelas de fatura e cobrança, permitindo a geração de QR Code para pagamento.

---

:star: **Fatura futura alimentação**

Foi criado um novo relatório para exibir a fatura futura dos clientes do segmento de alimentação, permitindo ao operador consultar os valores previstos antes do fechamento da fatura.

---

:arrow_up: **Implementado fluxo de criação de credenciada para criar Credenciada_ADM**

Agora, ao cadastrar ou editar uma Credenciada, o sistema verifica automaticamente se já existe um vínculo entre a Credenciada e a Administradora logada no iCRM4. Caso o vínculo não exista, ele é criado automaticamente com os seguintes dados:

- **AdmCodigo:** Código da Administradora logada
- **CrdCodigo:** Código da Credenciada Cadastrada/Editada
- **CrdAdmStatus:** Status do vínculo entre a Credenciada e a Administradora (`1 - ATIVO`)

> Solicitação **#4675**

---

:arrow_up: **Aplicado máscara no número do cartão no relatório de cobrança parcela**

Nos relatórios `RPT_CobrancaParcelasPRCT` e `RPT_CobrancaParcelasCMPL` foi aplicada máscara no campo de número do cartão para ocultação dos dígitos, respeitando os dois formatos:

- **16 dígitos:** `XXXX-XXXX-XXXX-XXXX`
- **19 dígitos:** `XXX-XXXX-XXXX-XXXX-XXXX`

---

:arrow_up: **Melhoria no filtro da página Impressão Expressa**

Os campos individuais de **Código do Cliente** e **CPF** foram removidos do filtro da página de Impressão Expressa. No lugar, foi adicionado um campo de **múltiplas pesquisas**, permitindo ao operador buscar pelo mesmo campo utilizando: nome, código ou CPF do cliente.

## :wrench: Configurações necessárias

**OBS1:** Necessário executar os scripts de banco de dados antes do deploy.

**OBS2:** Necessário atualizar o iCRM4 e o iCRM4Processa.

**OBS3:** Para habilitar a integração com o iBank, siga os passos abaixo:

---

### 1. Instalação do iBank

Escolha um diretório para instalação. Recomendamos:

```
C:\CRM\Account\ADMIN\WebservicesREST\iBank
```

Copie os executáveis **ibank-webapi.exe** e **ibank-webhook.exe** para o diretório escolhido.

No mesmo diretório, crie o arquivo **config.yaml** com o seguinte conteúdo, ajustando os valores conforme o ambiente do cliente:

```yaml
server:
  host: localhost
  port: 8010

database:
  type: oracle
  oracle:
    service: 127.1.1.1
    user: icrm
    password: 123

logging:
  level: debug
  format: console
  output_file: C:\CRM\Account\ADMIN\WebservicesREST\ibank\log\ibank.log
  max_size_mb: 1
  max_age_days: 1
  compress: false

providers:
  environment: production
```

Execute o **ibank-webapi.exe** e o **ibank-webhook.exe**. Os serviços serão registrados automaticamente no Gerenciador de Serviços do Windows como **iBank Web API** e **iBank WebHook Server**. Na primeira execução, o `ibank-webapi.exe` criará as tabelas necessárias no banco de dados e gerará o arquivo **apikey.txt** no mesmo diretório.

---

### 2. Configurar parâmetro do iBank no iCRM4

Acesse **Configurações > Parâmetros > ParametrosGerais** e siga os passos:

1. Clique em **Novo Grupo**, digite `IBANK` e clique em **Criar**.
2. Selecione o grupo **IBANK** e clique em **Visualizar** (ícone de lupa).
3. Clique em **Novo Parâmetro** e preencha os campos:

| Campo | Valor |
|---|---|
| **Apelido** | `API KEY IBANK` |
| **ID** | `PARIBANKXAPIKEY` |
| **Descrição** | `Conteúdo do arquivo apikey.txt gerado pelo iBank para autenticação de todas as comunicações internas` |
| **Tipo** | `Texto` |
| **Valor** | *(conteúdo do arquivo `apikey.txt` gerado na pasta de instalação do iBank)* |

---

### 3. Configurar proxy reverso no Apache (Webhook)

Para que o banco consiga enviar notificações ao **ibank-webhook**, é necessário configurar um **proxy reverso no Apache**, tornando a porta interna `8011` acessível pela URL pública.

---

### 4. Configurar Boleto e Fatura no iCRM4

Acesse **Configurações > Faturamento > Boleto e Fatura**.

Altere ou crie uma **"Configuração Boleto e Faturas"** e preencha os novos campos:

| Campo | Descrição |
|---|---|
| **É Integração?** | Marque para habilitar a integração. Exibe o grupo de campos **Integração**. |
| **Sistema:** | Selecione `Intellisys` |
| **Domínio:** | Endereço local da API do iBank (ex: `http://localhost:8010`) |
| **Client ID:** | `clientID` do cliente no webservice do banco |
| **Client Secret:** | `clientSecret` do cliente no webservice do banco |
| **API Key:** | `appKey` do cliente no webservice do banco |
| **Gerar Pix?** | Marque para adicionar PIX ao boleto |
| **Chave PIX:** | Chave PIX cadastrada no convênio do cliente com o banco |
| **WebHook:** | URL do webhook do WSICRM4REST — endpoint padrão: `{url}/webhook/ibank/retorno` |
| **Ativa o WebHook?** | Habilita ou desabilita o recebimento de notificações via webhook |
| **Emails Notificação:** | Lista de e-mails separados por `;` para notificações de erro nos boletos |

---

### 5. Vincular a Configuração ao módulo correspondente

Caso tenha criado uma **nova** configuração, é necessário vinculá-la ao módulo correto:

- **Cartão Próprio (Private):** Na bandeira, altere o campo **"Configuração de Boleto da Fatura do Cliente"**.
- **Convênio/Alimentação:** Em **Convênio/Alimentação > Conveniadas da Bandeira**, altere o campo **"Configuração de Boleto da Fatura do Convênio"**.
- **Cobrança:** Em **Configurações > Administradora > Cobrança**, altere o campo **"Configuração do Boleto da Fatura"**.

---

### 6. Verificar Eventos

Garanta que os **Eventos 15** e **18** estejam configurados e rodando corretamente.
