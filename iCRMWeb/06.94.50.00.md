## :wrench: Configurações necessárias

### Painel web do sistema iCRM Intellisys
O sistema utiliza um arquivo `config.js` no servidor para configuração em runtime, permitindo o mesmo build ser utilizado em diferentes ambientes sem necessidade de recompilação.

1. Arquivo `config.js` (obrigatório no servidor)

Criar o arquivo `config.js` na raiz do diretório de publicação. Um modelo está disponível em `config.example.js` gerado no build:

```` javascript
window.__ENV__ = {
  VITE_API_URL: "http://<HOST>:<PORTA>/api/v1",
  VITE_ICRMWEB_URL: "http://<HOST>/ICRM",
}
````

`VITE_API_URL` : Url do web service WSICRMWeb
`VITE_ICRMWEB_URL` : Endereço do site do ICRMWeb

2. JWT no iCRMWeb

Para o iCRMWeb gerar a chave JWT para o painel receber as informações do usuário logado, e dados do organizador, foi necessário desenvolver um programa `jwt.exe` que deve ficar no diretório raiz do site iCRMWeb. Esse programa será disponibilizado juntamente com a versão do Site.

3. WSICRMGO

O Back-end do PainelICRM é o novo webservice feito em GoLang, para o painel funcionar ele deve está instalado e configurado.

### WebSercice WSICRMGO
A configuração do WSICRMGO é feita atravez do arquivo `config.yaml`.

**Obrigatórias** — app recusa iniciar sem elas:

| config.yaml | Descrição |
|-------------|-----------|
| `auth.signing_key` | Chave secreta para assinatura JWT |
| `auth.issuer` | Identificador do emissor JWT |

**Banco de dados** — pelo menos o banco ativo precisa estar configurado:

| config.yaml | Descrição |
|-------------|-----------|
| `databases.active` | Banco ativo: `icrm3` ou `icrm4` |
| `databases.icrm4.host` | Host do Oracle |
| `databases.icrm4.service` | Service name do Oracle |
| `databases.icrm4.user` | Usuário do banco |
| `databases.icrm4.password` | Senha do banco |

Substitua `ICRM4` por `ICRM3` para configurar a instância icrm3.

**config.yaml**
```` yaml
server:
  port: 8080                    # porta do servidor HTTP

databases:
  active: icrm3                 # banco ativo: "icrm3" ou "icrm4"
  icrm3:
    host: ""                    # host Oracle
    port: 1521
    service: ""                 # service name Oracle
    user: ""
    password: ""
    max_open_conns: 10
    max_idle_conns: 5
  icrm4:
    host: ""
    port: 1521
    service: ""
    user: ""
    password: ""
    max_open_conns: 10
    max_idle_conns: 5

auth:
  signing_key: SuperI2304IcrM   # OBRIGATÓRIO : para geração do token de acesso
  issuer: WServICrmIntellsys    # OBRIGATÓRIO : para geração do token de acesso
  rate_limit_per_min: 30        # requisições/min por IP no /auth (0 = desabilitado)

logging:
  level: info                   # debug, info, warn, error
  format: console               # console (legível) ou json (estruturado)
  output_file: ""               # caminho do arquivo de log
  max_size_mb: 1                # tamanho máximo do arquivo antes de rotacionar
  max_age_days: 30              # dias para manter arquivos rotacionados
  max_backups: 0                # número máximo de backups (0 = ilimitado)
  compress: false               # comprimir arquivos rotacionados

````
